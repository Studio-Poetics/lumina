# Vernacular STEM Design Guidelines: The "Desi Logic" Approach

## 1. Core Philosophy
The goal is **Intuition over Terminology**. We are building a platform for the inquisitive mind that may not have English fluency. The language should not feel like a "Translation" of an English textbook, but like an original conversation happening in a local workshop.

**The Golden Rule:** If a 10-year-old or a non-English speaking mechanic cannot understand the sentence, rewrite it.

---

## 2. The Persona: "The Skilled Artisan" (Ustad)
Do not sound like a University Professor or a Computer.
Sound like a friendly, skilled **Artisan (Ustad)** or an **Elder Sibling**.

*   **Tone:** Encouraging, patient, authoritative but accessible.
*   **Vibe:** "Come, sit, let me show you how this machine works."
*   **Avoid:** Academic formality ("It is imperative to observe...").
*   **Use:** Direct conversational style ("See this? Watch what happens when we pull this lever...").

---

## 3. Vocabulary Strategy

### A. The "Hinglish" Reality (Tech Terms)
**Do not translate standard coding syntax into Hindi.**
Code is written in English. Translating "Variable" to "Char" (चर) confuses the student because the code editor still requires them to type `var` or `let`.

*   **Correct:** `Variable` (in Devanagari script: वेरिएबल)
*   **Incorrect:** चर (Char) or मान (Maan)

### B. The Bracket Rule (Definition in Flow)
When introducing a technical word, keep the English sound but immediately explain it in the vernacular using brackets.

*   **Format:** `Tech Word (Simple Desi Concept)`
*   **Example 1:** "Normalize (हिस्सेदारी / Batwara)"
*   **Example 2:** "Coordinates (पता / Address)"
*   **Example 3:** "Function (काम करने का तरीका / Recipe)"

### C. Everyday Hindi vs. Shuddh (Pure) Hindi
We use **Hindustani**—the spoken language of the streets, not the formal language of news anchors.

| Concept | ❌ Academic / Formal Hindi | ✅ Vernacular / Desi Hindi | Why? |
| :--- | :--- | :--- | :--- |
| **Logic** | तर्कशास्त्र (Tark-shastra) | लॉजिक (Logic) / दिमाग | "Tark-shastra" sounds scary. |
| **Observation** | अवलोकन (Avalokan) | देखो / ध्यान से देखो | "Avalokan" is a textbook word. |
| **Error** | त्रुटि (Truti) | गलती / गड़बड़ | "Gadbad" feels fixable. "Truti" feels like an exam fail. |
| **Change** | परिवर्तित करें (Parivartit karen) | बदलो (Badlo) | Keep verbs active and short. |

---

## 4. Analogy Framework
Abstract Math must be mapped to Physical Reality.

| Mathematical Concept | The Vernacular Analogy |
| :--- | :--- |
| **Grid (X, Y)** | **The City Map.** X is the Street number, Y is the House number. |
| **Sine Wave** | **Breathing (Saans) or Swing (Jhula).** It goes up, pauses, comes down, pauses. It has a rhythm. |
| **Time (t)** | **The Heartbeat.** It is the clock that never stops ticking. |
| **Normalization (0.0 - 1.0)** | **The Dimmer Switch.** 0 is off, 1 is full on. It's a percentage. |
| **Variables** | **Containers (Dabbe).** A box named 'speed' where we keep a number. |

## 5. UI Text Guidelines

### Buttons & Labels
Labels should describe the **outcome**, not the mechanical function.

*   **Run:** `चलाओ` (Run it) instead of `निष्पादित करें` (Execute).
*   **Explain:** `समझाओ` (Explain to me) instead of `विवरण` (Description).
*   **Next:** `आगे बढ़ें` (Move forward) instead of `अगला` (Next).

### Error Messages
Never blame the user. Treat errors as "loose wires" or "confused instructions."

*   **Bad:** "Syntax Error at Line 5."
*   **Good:** "लाइन 5 में कुछ समझ नहीं आया। क्या कोई ब्रैकेट छूट गया है?" (Didn't understand Line 5. Is a bracket missing?)

---

## 6. Implementation Checklist
Before finalizing any Hindi text, ask:
1.  Is this how I would say it to my neighbor?
2.  Did I remove all Sanskritized/Formal words?
3.  Are the technical terms preserved phonetically?
4.  Is the tone encouraging?

> **"Observe the pattern. Trust the logic."**
> **"पैटर्न को देखो। लॉजिक पर भरोसा करो।"**
